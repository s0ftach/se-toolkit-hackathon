📋 Context

End User

Students and young people who go to the gym regularly and want to track their progress over time.

Problem

People track workouts in notes or in their head and lose progress history — they can't see if they're actually improving.

Solution

A web application where users log exercises via a simple form, and view their personal progress with interactive charts.

⚙️ Implementation

Version 1

Core feature — log workouts & view progress

User registration/login with JWT auth, log exercise sets (exercise, weight, reps), view workout history table, view progress chart (weight over time).

Stack: FastAPI, SQLite, Chart.js, vanilla HTML/CSS/JS, Docker Compose.

Version 2

Improvements & polish

Added reps bar chart alongside weight line chart, today's workout summary, responsive mobile-friendly design, single-container deployment.

TA feedback addressed: removed Telegram bot dependency (blocked on university VMs), made web app the primary client, simplified deployment to single container.

Backend

FastAPI + aiosqlite
JWT authentication
REST API
Lazy DB init

Database

SQLite 3 (ACID)
Users + Sets tables
Docker volume persistence

Frontend

Vanilla HTML/CSS/JS
Chart.js 4
3-tab interface
Responsive design
